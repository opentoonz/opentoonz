# Internationalization (i18n)

## Overview

Language switching uses Qt's `QTranslator` system with hot-switch support. Translations are `.ts` source files compiled to `.qm` binary files.

## Supported Languages

| Display Name | Code | File |
|-------------|------|------|
| English | (built-in, no translator needed) | — |
| 中文 | `chinese` | `translations/chinese/template.ts` |
| 日本語 | `japanese` | `translations/japanese/template.ts` |

## File Format (Qt TS XML)

```xml
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE TS>
<TS version="2.1" language="zh_CN">
<context>
    <name>StackedMenuBar</name>    <!-- QObject class name = tr() context -->
    <message>
        <source>File</source>
        <translation>文件</translation>
    </message>
    <message>
        <source>Settings</source>
        <translation>设置</translation>
    </message>
</context>
</TS>
```

**Important:** The `<name>` must match the C++ class where `tr()` is called. Qt uses the class name as the translation context.

## Adding a New Language

### Step 1: Create .ts file

```bash
mkdir translations/korean
```

Create `translations/korean/template.ts`:

```xml
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE TS>
<TS version="2.1" language="ko_KR">
<context>
    <name>StackedMenuBar</name>
    <message>
        <source>File</source>
        <translation>파일</translation>
    </message>
    <!-- ... add all translatable strings ... -->
</context>
</TS>
```

### Step 2: Compile .qm

```bash
lrelease translations/korean/template.ts -qm translations/korean/template.qm
```

### Step 3: Register in AppContext

In `appcontext.cpp`, add the language to `m_availableLanguages`:

```cpp
m_availableLanguages << "English" << "中文" << "日本語" << "한국어";
```

And in `languageCode()`:

```cpp
if (m_currentLanguage == "한국어") return "korean";
```

### Step 4: Update translations.qrc

```xml
<file alias="korean/template.qm">../translations/korean/template.qm</file>
```

### Step 5: Copy .qm to runtime

```bash
mkdir -p runtime/translations/korean
cp translations/korean/template.qm runtime/translations/korean/
```

## How Translation Works

### Loading

In `AppContext::loadTranslator()`:

```cpp
QTranslator* t = new QTranslator(this);
// Try filesystem first (runtime/translations/<code>/template.qm)
bool ok = t->load(fsPath);
if (!ok) {
    // Fallback: load from Qt resource
    QFile qmFile(resPath);
    QByteArray data = qmFile.readAll();
    ok = t->load(data.constData(), data.size());
}
if (ok) qApp->installTranslator(t);
```

### Hot-Switching

When the user selects a new language:

```cpp
AppContext::setCurrentLanguage("中文")
  ├── unloadTranslator()            // removeTranslator + delete old
  ├── loadTranslator("chinese")     // new translator
  ├── saveSettings()                // persist to settings.xml
  └── emit languageChanged()        // triggers menu rebuild
        └── StackedMenuBar::refreshCurrentMenu()
             └── buildDefaultMenuBar() → all tr() calls use new translator
```

### Context Resolution

```cpp
// In StackedMenuBar::buildDefaultMenuBar():
tr("File")    // context = "StackedMenuBar" (the class name)

// In RoomTabWidget::contextMenuEvent():
tr("New Room")  // context = "RoomTabWidget"

// In MainWindow::defineActions():
tr("Save Layout")  // context = "MainWindow"
```

## Using lupdate to Sync Translations

```bash
lupdate src/ -ts translations/chinese/template.ts
lupdate src/ -ts translations/japanese/template.ts
```

This scans source files for `tr()` calls and updates the `.ts` files with new/changed strings.
